# 谷歌开源社区行为准则

在谷歌，我们尊重并庆祝开源贡献者的创造力与协作精神，尊重他们带到项目与社区的多样技能、经验、文化和观点。

所有谷歌的开源项目和社区都应是包容性的环境，基于对所有人的尊重，不因性别认同与表达、性取向、残疾、神经多样性、外貌、体型、族裔、国籍、种族、年龄、宗教或类似的个人特征而区别对待。

我们重视多样化的观点，但我们更重视尊重的行为。

尊重的行为包括：

- 体谅、友善、建设性且乐于助人。
- 不使用贬低、歧视、骚扰、仇恨、性化或身体威胁相关的行为、言语或图片。
- 不进行不受欢迎的身体接触。

一些谷歌的开源项目可能采用具体的项目行为守则（project code of conduct），这些守则可能包含对参与者的额外期望。大多数此类项目将使用我们修改过的 Contributor Covenant。

## 和平解决冲突

我们认为并非所有冲突都是坏事；健康的争论与分歧常常带来积极结果。但绝不应以不尊重的方式行事。

如果你发现有人行为不当，建议你直接与相关人员沟通。许多问题可以迅速并轻易地解决，这也让当事人对结果有更多控制权。如果你无法解决此事，或行为具有威胁性或骚扰性质，请报告该事件。我们致力于提供一个让参与者感到受欢迎和安全的环境。

## 举报问题

某些谷歌的开源项目可能采用项目特定的行为准则。在这些情况下，会指定一名谷歌员工作为项目管理者（Project Steward），负责接收和处理行为准则违规举报。如果项目尚未指定 Project Steward，你可以通过发送邮件至 opensource@google.com 进行举报。

我们会调查每一份投诉，但你可能不会收到直接回复。我们将酌情决定是否以及如何对举报进行后续处理，后续行动可能包括不采取措施到永久性逐出项目及其项目关联空间。在采取任何行动前，我们会通知被举报人并给予其说明与讨论的机会。举报者的身份将在提供给被举报人的报告细节中被省略。在潜在危及人身安全或存在持续骚扰等有害情形时，我们可能会在不事先通知的情况下采取行动。

*该文档改编自 IndieWeb 的行为准则，并可在 <https://opensource.google/conduct/> 找到。*
